## 📖 درباره پروژه

**IP2** یه پروژه‌ی خفن و تمام‌اتوماتیک برای پیدا کردن **آی‌پی‌های تمیز و زنده** با استفاده از زیرساخت رایگان **GitHub Actions** هست. این اسکنر با **۱۷ شارد** (هر شارد = یه رانر مجزا) به‌مدت **۶ ساعت تمام** موازی کار می‌کنه، از **۱۰ منبع آنلاین پروکسی** تغذیه می‌شه، میلیون‌ها آی‌پی تصادفی جنریت می‌کنه، و هر آی‌پی رو با **پینگ واقعی (ICMP)** و **تست TCP** بررسی می‌کنه تا فقط آی‌پی‌های واقعاً زنده رو تحویلت بده.

## 🚀 ویژگی‌های اصلی

| ویژگی | توضیح |
|-------|--------|
| 🧩 **۱۷ شارد هم‌زمان** | ۱۷ رانر موازی که همگی باهم کار می‌کنن |
| ⏱️ **۶ ساعت اسکن مداوم** | هر شارد ۵.۹۲ ساعت بدون توقف اسکن می‌کنه |
| 🧵 **۵۰۰ نخ در هر شارد** | تست هم‌زمان ۵۰۰ آی‌پی در هر رانر |
| 🌐 **فید از ۱۰ منبع آنلاین** | لیست پروکسی‌های معروف (ProxyScrape, GitHub, Proxy-List) |
| 🎲 **جنریت تصادفی + seed** | آی‌پی‌های Public تصادفی + گسترش subnetهای معتبر |
| 📡 **پینگ واقعی (ICMP)** | تست ping واقعی قبل از TCP (قابل غیرفعال‌سازی) |
| 🔌 **تست TCP روی پورت دلخواه** | پیش‌فرض پورت 80، قابل تنظیم روی هر پورت |
| 📦 **خروجی پارتیشن‌بندی‌شده** | فایل‌های ۵۰۰KB با شماره‌گذاری خودکار |
| 🔄 **به‌روزرسانی زنده** | هر ۱۰ ثانیه نتایج توی ریپو کامیت می‌شن |
| 🧹 **حذف آی‌پی تکراری** | خروجی نهایی کاملاً یکتاسازی شده |

### فازهای اسکن:
1. **فاز ۱ – جمع‌آوری از وب**: ۱۰ منبع معروف پروکسی HTTP/HTTPS اسکرپ می‌شن
2. **فاز ۲ – جنریت**: آی‌پی‌های تصادفی Public + همسایه‌های seed IPها
3. **تست**: پینگ واقعی (ICMP) → تست TCP Connect → ذخیره‌ی آی‌پی‌های زنده

## 📁 فایل‌های خروجی

| فایل | توضیح |
|------|-------|
| `live_ips_01.txt`, `live_ips_02.txt`, ... | آی‌پی‌های زنده (هر ۲۰ ثانیه به‌روز می‌شن) |
| `live_ips_with_port_01.txt`, ... | آی‌پی:پورت (همراه با شماره پورت) |
| `white_ips_01.txt`, `white_ips_02.txt`, ... | خروجی نهایی تمیز بعد از پایان اسکن |
| `white_ips_with_port_01.txt`, ... | خروجی نهایی با پورت |
| `shard_X_ips.txt` | فایل‌های موقت هر شارد (داخلی) |

> ⚠️ **توجه**: هر فایل حداکثر **۵۰۰ کیلوبایت** حجم داره. با رشد آی‌پی‌ها، خودکار فایل‌های جدید (live_ips_02، live_ips_03 و ...) ساخته می‌شن.

## ⚙️ راه‌اندازی

### ۱. تنظیم Secret

قبل از هر چیزی، یه **Personal Access Token (Classic)** با دسترسی `repo` بساز:

- برو به **Settings > Developer settings > Personal access tokens > Tokens (classic)**
- روی **Generate new token (classic)** کلیک کن
- اسم بذار (مثلاً `IP2-SCANNER`)
- اسکوپ **`repo`** رو تیک بزن
- **Generate token** رو بزن و کپیش کن

حالا توی ریپو:
- برو به **Settings > Secrets and variables > Actions**
- روی **New repository secret** کلیک کن
- اسم: `GH_PAT`
- مقدار: توکنی که کپی کردی
- **Add secret** رو بزن

### ۲. فایل seed_ips.txt (اختیاری ولی توصیه می‌شه)

تو ریشه‌ی ریپو یه فایل `seed_ips.txt` بساز و چندتا آی‌پی معتبر توش بذار (هر خط یه آی‌پی):

این آی‌پی‌ها رو اسکنر به‌عنوان نقطه‌ی شروع استفاده می‌کنه و کل subnet اون‌ها رو اسکن می‌کنه (یعنی ۲۵۶ آی‌پی برای هر seed).

### ۳. اجرای اسکنر

- برو به تب **Actions** ریپو
- از لیست workflowها، **⚡ Infinite IP Scanner — 20 Shards, 6‑Hour Marathon** رو انتخاب کن
- روی **Run workflow** کلیک کن
- پارامترها رو تنظیم کن:

| پارامتر | مقدار پیش‌فرض | توضیح |
|---------|---------------|-------|
| `port` | `80` | پورت TCP برای تست (80=HTTP, 443=HTTPS, 22=SSH, ...) |
| `timeout` | `10` | حداکثر زمان انتظار برای اتصال TCP (ثانیه) |
| `skip_ping` | `false` | `true` = پینگ ICMP رو رد کن (۳برابر سریع‌تر، ولی دقت کمتر) |

- روی **Run workflow** کلیک کن

### ۴. مشاهده‌ی نتایج

- بلافاصله بعد از شروع، فایل‌های `shard_X_ips.txt` توی ریپو ظاهر می‌شن
- هر ۲۰ ثانیه، فایل‌های `live_ips_01.txt`، `live_ips_02.txt` و ... به‌روز می‌شن
- بعد از پایان ۶ ساعت، فایل‌های نهایی `white_ips_01.txt` و ... جایگزین می‌شن
- تب **Commits** رو ببینید تا رشد آی‌پی‌ها رو لحظه‌ای دنبال کنید

## 💡 تنظیمات پیشنهادی برای سناریوهای مختلف

| هدف | port | timeout | skip_ping |
|-----|------|---------|-----------|
| 🏎️ **حداکثر سرعت** (پیدا کردن IP وب‌سایت) | 80 | 2 | `true` |
| 🎯 **دقت بالا** (IPهای واقعاً زنده) | 80 | 10 | `false` |
| 🔒 **IPهای HTTPS** | 443 | 5 | `false` |
| 🖥️ **SSH Serverها** | 22 | 5 | `false` |
| 🌐 **پیدا کردن Proxy** | 8080 یا 3128 | 3 | `true` |

## 🧠 نکات فنی

- **محدودیت گیت‌هاب**: حداکثر ۲۰ job هم‌زمان برای مخازن عمومی. در این پروژه ۱۷ شارد استفاده شده تا فضای کافی برای jobهای `merge` و `live-merge` باقی بمونه.
- **عمر رانر**: هر شارد `timeout-minutes: 350` (حدود ۵.۸۳ ساعت) تنظیم شده. این یعنی از ۶ ساعت مجاز گیت‌هاب نهایت استفاده رو می‌بره.
- **نرخ یافتن آی‌پی**: با skip_ping=true، هر شارد حدود ۱۵۰–۳۰۰ هزار تست TCP در ۱۰ دقیقه انجام میده. درصد آی‌پی زنده معمولاً ۱–۳٪ آی‌پی‌های تصادفیه.
- **seed_ips.txt**: هر seed باعث اسکن ۲۵۶ آی‌پی (کل subnet) می‌شه. اگه seed معتبر باشه، شانس پیدا کردن آی‌پی زنده توی subnetش خیلی بالاست.
- **فضای دیسک**: هر فایل live_ips حداکثر ۵۰۰KB هست. با ۱۰۰ فایل، حدود ۵۰MB فضا اشغال می‌شه که توی محدوده‌ی مجاز ریپوی رایگان گیت‌هاب (۱GB) قرار داره.
